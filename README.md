# Система бронирования ресурсов

## Описание проекта

Данный веб‑сайт представляет собой систему бронирования ресурсов (переговорные комнаты, оборудование и др.) с разграничением прав доступа пользователей и администраторов.

Проект состоит из:
- клиентской части (React)
- серверной части (Node.js + Express)
- базы данных (PostgreSQL)

---

## Роли пользователей

### Пользователь
- просмотр доступных ресурсов
- создание бронирований
- просмотр и управление своими бронированиями

### Администратор
- все возможности пользователя
- управление ресурсами
- управление пользователями
- просмотр логов системы

Административные разделы автоматически скрыты для обычных пользователей.

---

## Авторизация и регистрация

### Вход
1. Перейдите на страницу входа
2. Введите логин и пароль
3. Нажмите кнопку **Войти**

При успешной авторизации пользователь перенаправляется на главную страницу.

### Регистрация
1. Перейдите на страницу регистрации
2. Заполните обязательные поля
3. Нажмите **Зарегистрироваться**

По умолчанию новый пользователь получает роль **user**.

---

## Главная страница (Dashboard)

На главной странице отображаются карточки действий:
- Ресурсы
- Мои бронирования
- Администрирование (только для admin)

Карточки:
- кликабельны
- выровнены по центру
- содержат SVG‑иконки

Нажатие на карточку выполняет переход в соответствующий раздел.

---

## Ресурсы

В разделе ресурсов пользователь может:
- просматривать список доступных ресурсов
- видеть описание ресурса
- создавать бронирование (если ресурс доступен)

Администратор дополнительно может:
- добавлять ресурсы
- редактировать ресурсы
- удалять ресурсы

---

## Бронирования

### Создание бронирования
1. Выберите ресурс
2. Укажите дату и время
3. Подтвердите бронирование

### Мои бронирования
В разделе отображается:
- список бронирований пользователя
- дата и время
- связанный ресурс

Пользователь может отменять свои бронирования.

---

## Администрирование

Раздел доступен **только администраторам**.

### Пользователи
Администратор может:
- просматривать список пользователей
- создавать новых пользователей
- удалять пользователей
- изменять роль (user / admin)
- блокировать и разблокировать пользователей

### Логи
В логах фиксируются действия:
- вход в систему
- создание / обновление / удаление данных

Логи отсортированы по дате (новые сверху).

## Запуск проекта

### Клиент

```bash
npm install
npm start
```

Приложение будет доступно по адресу:
```
http://localhost:3001
```

### Сервер

```bash
npm install
npm run dev
```

API доступно по адресу:
```
http://localhost:3000/api
```
